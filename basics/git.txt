git init - инициализирует пустой локальный репозиторий, понять инициализирован ли локальный репозиторий можно при помощи команды ls -a (если создан то покажет файл .git)

git add . (либо файл/директория вместо '.') - добавляет все файлы/директории в индекс гита (отслеживает все файлы/директории)

git commit -m '<сообщения для коммита>' - создает новую версию, если были были изменения в отслеживающихся файлах/директориях

git remote add <origin> SSH-link/HTTPS-link - устанавливает связь с удаленным репозиторием на github

git push origin <название ветки> - отправляет все изменения которые сохранил в коммите на удаленный репозиторий

git clone HTTPS-link/SSH-link - клонирует проект из удаленного репозитория на github на ваш ПК (локальный репозиторий)

git branch - показывает все ветки ветки которые есть, и на какой ветке вы  сейчас находитесь
git branch название_ветки - создает новую ветку по указанному названию

git checkout <название ветки> - переходит на другую ветку


1) git init - инициализирует локальный репозиторий
2) git add . - добавляет все файлы/директории в индекс гита
3) git commit -m 'комментарий' - коммитит изменения
4) ssh-keygen - генерирует ssh ключ (используется только если до этого к гитхабу не привязывали этот ключ)
    4.1) переходим в директорию .ssh (путь до .ssh можно посмотреть при генерации ключа)
    4.2) cat id_rsa.pub - выводим в терминал сгенерированный ssh ключ
    4.3) переходим на github в раздел setting, ssh and gpg keys и добавляем наш скопированный ключ
    4.4) переходим обратно в рабочую директорию
5) git remote add origin SSH-link/HTTPS-link
6) git push origin main/master - отправляем все сохраненные файлы/директории на github
